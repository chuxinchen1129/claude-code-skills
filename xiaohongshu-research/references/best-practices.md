# 小红书调研技能 - 最佳实践

**文档版本**：v1.0.0
**更新时间**：2026-03-02

---

## 命名规范

### 技能命名
- 使用小写字母+连字符+数字：`xiaohongshu-research`
- 避免使用 `helper`, `utils`, `tools` 等通用名称
- 推荐使用 `research`, `collector`, `analyzer` 等描述性名称

### Python 文件命名
- 使用小写字母+下划线：`main.py`, `crawler.py`, `processor.py`
- 避免中文文件名（确保跨平台兼容性）

---

## 代码风格指南

### 简洁至上
- 优先使用已存在的库和函数，避免重复造轮子
- 代码长度控制在合理范围，单个文件不超过500行
- 避免过度抽象，保持代码直观易懂

### 文档字符串
- 使用描述性文档字符串（docstring）
- 清晰说明函数的输入和输出
- 包含使用示例

### 错误处理
- 使用 try-except 捕获和处理异常
- 提供清晰的错误信息
- 在main函数中统一处理 KeyboardInterrupt

---

## MediaCrawler 集成注意事项

### 配置修改
- 修改配置前先备份原文件
- 修改后立即恢复默认值（或使用临时配置）
- 使用正则表达式精确替换

### 缓存管理
- 每次采集前检查是否需要清空
- 清空后验证目录结构是否正确
- 避免混入旧数据导致分析结果不准确

---

## 数据处理最佳实践

### 互动分计算
- 使用统一的计算公式：点赞 + 收藏×2 + 评论×3 + 分享×4
- 处理"万"单位（1万=10000）
- 处理空值和异常格式

### 数据筛选
- 基于关键词的文本匹配（标题或描述包含关键词）
- 去重时使用 note_id 唯一键
- 保留原始数据，筛选后生成新数据集

---

## 文件路径处理

### 跨平台兼容
- 使用 pathlib.Path 替代 os.path
- 使用 `/` 拼接路径（pathlib自动处理）
- 避免硬编码路径分隔符

### 相对路径与绝对路径
- Skill 内部使用相对路径
- 用户输出路径使用绝对路径
- 临时文件使用系统临时目录

## 测试建议

### 单元测试
- 核心函数编写单元测试（可选）
- 测试边界情况（空数据、异常输入）
- 使用 pytest 或 unittest 框架

### 集成测试
- 端到端测试关键流程
- 验证 MediaCrawler 调用是否正确
- 验证文件输出是否符合预期

---

## 性能优化

### 大数据量处理
- 对于超过1000条数据，使用分块处理
- 避免在内存中加载全部数据

### 缓存策略
- 使用 MediaCrawler 的 JSON 缓存机制
- 避免重复采集相同数据

---

## 安全注意事项

### 敏感信息
- 不在代码中硬编码密码、Token等敏感信息
- 使用环境变量或配置文件管理

### 文件权限
- 避免创建可执行文件
- 设置适当的文件权限（644）
