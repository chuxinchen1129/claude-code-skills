# 写作风格标准化器 v2.0

> **核心定位**：所有写作技能的风格中枢，确保输出内容符合统一的品牌调性和写作规范。
>
> **双模式**：
> - **风格学习模式**：参考 samples/ 中的范文，学习并模仿其风格
> - **风格检查模式**：对照标准检查清单，验证内容是否符合规范

---

## 工作模式

### 模式1：风格学习（Style Learning）

**触发方式**：
- "参考XXX的风格来写"
- "学习sample_06的写法"
- "用品牌故事的风格写"

**执行流程**：
1. 读取对应 sample 的内容
2. 提取风格特征（结构、语气、关键词）
3. 应用到新内容生成
4. 保持风格一致性

### 模式2：风格检查（Style Checking）

**触发方式**：
- "检查这篇文章的风格"
- "是否符合风格标准"
- "风格检查"

**执行流程**：
1. 读取生成的内容
2. 对照风格检查清单逐项验证
3. 标记不符合规范的部分
4. 提供修改建议

## 风格模式定义

### 模式1：通用科普风格

**来源参考**：sample_07_sleep_science.md

**风格特征**：
- **语气**：客观中立、专业但不晦涩
- **结构**：
  1. 开篇：引发共鸣的问题
  2. 主体：清单式知识点（10个左右）
  3. 结尾：总结+软性CTA
- **语言**：
  - 数据和事实优先
  - 用类比降低理解门槛
  - 冷知识标签增加趣味
- **禁止**：
  - 夸张表达 ("绝绝子"、"神级")
  - 情绪化词汇
  - 未经验证的断言

**结构模板**：
```markdown
# 标题：[现象]背后的[科学原理]

## 引入（10%）
用常见现象或问题引入，引发读者共鸣
"你是不是也有过这样的经历——"

## 核心知识（70%）
01 [知识点1]
- 现象描述
- 科学原理
- 冷知识

02 [知识点2]
...

## 总结+CTA（20%）
一句话总结+软性行动召唤
"想要XXX？搜「XXX」试试看"
```

### 模式2：品牌推广风格

**来源参考**：sample_06_brand_story.md, sample_08_sleep_breakdown.md

**风格特征**：
- **语气**：温暖、专业、克制
- **核心原则**：
  - 软性植入，非硬广
  - 痛点驱动，解决方案自然引入
  - 科学背书，数据支撑
- **结构**：
  1. 痛点场景（引发共鸣）
  2. 科学原理（建立信任）
  3. 产品价值（自然呈现）
  4. 行动召唤（轻量 CTA）

**禁止词库**：
```
绝对禁止：
- 第一、最强、唯一
- 根治、彻底、100%
- 夸类形容词：神级、绝绝子、yyds

慎用：
- 感叹号（最多1处）
- 表情符号（仅用于品牌账号）
- 营销号式标题党
```

**品牌调性规范**（以悟昕为例）：
```yaml
关键词:
  - 科学循证
  - 温暖治愈
  - 优雅克制

语态:
  - 第三人称为主
  - 避免过度煽情
  - 用数据说话

卖点植入原则:
  - 超软性：提及而非推荐
  - 场景化：在解决方案中自然出现
  - 一次一处：不要堆砌
```

### 模式3：商业分析风格

**来源参考**：sample_01_whole_foods.md ~ sample_05_time_relativity.md

**风格特征**：
- **语气**：理性、深度、洞见驱动
- **结构**：
  1. 现象/背景（10%）
  2. 核心分析（60%）
     - 多维度拆解
     - 数据支撑
     - 对比分析
  3. 洞见提炼（25%）
  4. 趋势预判（5%）

**写作要点**：
- 用数据和事实说话
- 避免主观臆断
- 提供"为什么"而不仅是"是什么"
- 对比分析增强说服力

## 商业分析风格 · 核心创作方针

> **适用场景**：品类分析、商业模式、行业洞察、深度案例研究
> **核心理念**：让读者跟着你的思路走，而不是把观点硬塞给他们

---

### 一、叙事结构原则

#### 1. "从已知到未知"的叙事逻辑

**✅ 正确做法**：
- 从读者熟悉的事实入手（Comif 2000年 → 森永 2026年）
- 先讲历史脉络，再讲当下趋势，最后讲挑战和问题

**❌ 避免**：
- 一上来就抛出新概念或未经铺垫的观点

#### 2. 用问题引导思考，而非直接陈述

**✅ 正确做法**：
- "为什么要避开自己的核心优势？"
- "但中美日三国的节奏很不一样"
- 提出问题 → 分析原因 → 给出答案

**❌ 避免**：
- 平铺直叙地罗列事实

#### 3. 每个段落都要有"钩子"

**✅ 正确做法**：
- "这里有个耐人寻味的细节"
- "但你注意到了吗？"
- 用反问、转折、对比来保持读者注意力

**❌ 避免**：
- 段落开头就给结论

---

### 二、细节与证据

#### 4. 细节 > 抽象描述

**❌ 错误**："森永的定价策略很聪明"

**✅ 正确**："部分商品356日元（约17元人民币）。HI-CHEW软糖约150日元，普通宠物零食约300-500日元。森永的定价卡在中间"

**原则**：用具体数字、产品名称、定价来支撑观点

#### 5. 证据必须有来源，不能编造

**❌ 错误**："这样做是为了转移信任"（无证据的推论）

**✅ 正确**："这不仅是产品策略，更是在强化一个场景——主人和宠物能真正'一同分享'同一份零食"

**原则**：观点必须有事实支撑，不确定的就不要写

#### 6. 事实必须核实，尤其是时间、数据、案例

**✅ 实践案例**：
- 醇粹案例：原稿写2024年，核实官网后改为2019年
- 中国宠物市场数据：查Wind数据统计

**原则**：涉及时间、数据、品牌案例的，必须找到原始来源

---

### 三、过渡与连贯

#### 7. 段落之间必须有"桥梁"

**✅ 正确做法**：
> "这个小插曲暴露了一个现实：即便产品做好了，合规和渠道的挑战依然真实"

从具体案例（醇粹）自然过渡到普遍问题（监管困境）

**原则**：不要让读者在不同主题之间突兀跳跃

#### 8. 章节之间的承接要"拉远视角"

**✅ 正确做法**：
> "从2000年Comif的孤军奋战，到今天森永的入局，这条探索之路走了24年"

从具体品牌案例，拉回到整个行业的发展脉络

**原则**：宏观（行业趋势）↔ 微观（具体案例）交替进行

---

### 四、写作技巧

#### 9. "场景化" > "概念化"

**❌ 错误**："通过人类版和狗狗版同时售卖来建立信任转移"

**✅ 正确**："这不仅是产品策略，更是在强化一个场景——主人和宠物能真正'一同分享'同一份零食"

**原则**：描述场景，让读者自己感受，而不是直接给出理论分析

#### 10. 反向教育的高明之处

**✅ 实践案例**：
- "人能吃≠狗就安全" —— 表面提醒风险，实际强化必要性
- 不是说"我的产品有多好"，而是说"其他产品可能不安全"

**原则**：有时候"反向"比"直接"更有说服力

#### 11. 表格对比 vs 文字描述

**❌ 错误**："两条路径，两种逻辑"后面直接跟表格

**✅ 正确**：用文字清晰描述两条路径的区别和逻辑

**原则**：
- 表格适合快速查阅
- 文字适合叙事连贯
- 根据文章目的选择

---

### 五、修改流程

#### 12. 修改时的思考顺序

1. **叙事逻辑**：整体结构是否合理？是否符合"从已知到未知"？
2. **证据支撑**：每个观点是否有事实支撑？推论是否合理？
3. **过渡连贯**：段落、章节之间是否有自然衔接？
4. **事实核查**：时间、数据、案例是否准确？
5. **语言细节**：表达是否精准？是否有更好的说法？

#### 13. 用户反馈的优先级

| 优先级 | 反馈类型 | 处理方式 |
|--------|---------|---------|
| ⭐⭐⭐ | 质疑推论来源 | 立即承认错误并提供替代方案 |
| ⭐⭐⭐ | 要求核实事实 | 查找原始来源，更正错误 |
| ⭐⭐ | 要求更多细节 | 补充具体数据、产品名称、定价 |
| ⭐ | 叙事结构调整 | 重新组织章节顺序，添加过渡 |

---

### 六、质量检查清单

**发布前检查**：

- [ ] 所有推论都有证据支撑
- [ ] 时间、数据、案例已核实
- [ ] 段落之间有过渡句

- [ ] 用问题引导思考，而非平铺直叙

## 统一规范

### 标题规范

**科普类**：
- 格式：`[主题]+[疑问/亮点]`
- 示例："午睡超过这个时间，你可能越睡越累"
- 长度：15-25字

**品牌推广类**：
- 格式：`[痛点]+[解决方案暗示]`
- 示例："睡不够8小时？你可能睡错了重点"
- 长度：15-25字

**商业分析类**：
- 格式：`[主体]+[核心观点]`
- 示例："Whole Foods：为什么40%高价仍有800万周客"
- 长度：20-30字

### 开篇引入规范

**禁止**：
```
❌ "大家好，我是XXX"
❌ "今天我们来聊聊XXX"
❌ "最近XXX很火，我也来蹭个热度"
```

**推荐**：
```
✅ 场景化引入："你是不是也有过这样的经历——"
✅ 数据引入："据《XXX报告》显示，XX%的人..."
✅ 现象引入："有一个奇怪的现象：XXX"
```

### 段落规范

- **长度**：每段 80-120 字
- **密度**：手机屏幕 2-3 屏一个重点
- **分段**：每个观点独立成段
- **空行**：段落间空一行，视觉呼吸

### 引用规范

**数据引用**：
```
格式：来源+具体数字
示例："据《2025中国睡眠健康调查报告》显示，中国成年人失眠发生率高达48%。"
```

**专家引用**：
```
格式：专家头衔+姓名+观点
示例："斯坦福睡眠中心脑神经专家Andrew Huberman指出：'虽然我们每天都在做这件事，但90%的人其实并不懂它。'"
```

## 风格检查清单

使用本 skill 时，对照以下清单逐项检查：

### 内容层面
- [ ] 标题符合对应模式的长度和格式要求
- [ ] 开篇引入避免了无效铺垫
- [ ] 每个论点有数据或事实支撑
- [ ] 避免了禁止词汇和表达
- [ ] 品牌植入符合"超软性"原则（如适用）

### 结构层面
- [ ] 段落长度适中（80-120字）
- [ ] 重点内容有视觉突出（加粗/引用）
- [ ] 段落间有空行，易于阅读
- [ ] 层次结构清晰（小标题/序号）

### 语言层面
- [ ] 避免了网络流行语（除非特定风格需要）
- [ ] 专有名词有解释或类比
- [ ] 数字使用了阿拉伯数字
- [ ] 标点符号使用规范

## 调用接口

### 写作 skill 集成方式

在写作 skill 的 SKILL.md 中添加：

```markdown
## 风格控制

生成内容后，调用 `writing-style-standardizer` 进行风格统一：

**触发方式**：
1. 自动调用：在输出前自动进行风格检查
2. 手动触发：用户明确要求"统一风格"、"检查风格"

**参数**：
- 风格模式：科普风格 / 品牌推广 / 商业分析
- 参考样本：sample_XX（可选）
- 品牌调性：（可选）指定品牌规范
- 检查级别：基础 / 严格
```
