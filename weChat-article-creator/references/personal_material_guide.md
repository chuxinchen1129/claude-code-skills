## 🎯 核心理念

**问题**：AI 生成的内容容易有"AI 味"，缺乏真实感和可信度

**解决方案**：使用你在 Get 笔记中记录的真实经历、观点、案例

**效果**：
- ✅ 开头更自然：用真实经历引入
- ✅ 观点更可信：用真实评价支撑
- ✅ 案例更生动：用真实项目展示
- ✅ AI 味大幅降低：从 60% 降到 30% 以下

## 📱 Get 笔记素材收集流程

### 第一步：手机端记录（Get 笔记 App）

**支持 4 种记录方式**：

1. **文字记录**：直接输入文字
2. **AI 语音记录**：按住说话，AI 自动转文字 + 智能分段
3. **AI 图片记录**：拍照，AI 自动识别图片中的文字
4. **AI 链接记录**：分享链接，AI 自动提取正文内容

**记录格式建议**：

```markdown
你的经历/观点/案例

#标签1 #标签2
```

**实际示例**：

```markdown
今天用Claude Code写了个脚本，从需求到完成只用了30分钟。
感觉比ChatGPT方便，因为它能看到我的项目文件。

#AI编程工具 #开发工具
```

```markdown
悟昕资料整理项目完成了，花了2小时。
最有用的是品牌档案这个模块。

#悟昕 #项目复盘 #品牌档案
```

### 第二步：电脑端同步

**自动同步**：
```bash
cd ~/Desktop/大秘书系统/scripts
python3 sync_get_notes_final.py
```

**同步频率建议**：
- 轻度使用：每周一次

- 重度使用：每天一次

**同步位置**：

## ✍️ 写作时如何使用

### 场景 1：文章开头引入

**目标**：用真实经历作为文章开头，增加代入感

**操作**：
```
你："写一篇关于Claude Code的评测，开头要真实"

```

**AI 自动执行**：
1. 在 `~/Desktop/个人素材库/主题索引/` 中搜索 "AI编程工具"
2. 读取相关笔记
3. 提取真实经历："今天用Claude Code写了个脚本，从需求到完成只用了30分钟"

```markdown

最明显的是速度：从需求到完成只用了30分钟。
```

### 场景 2：观点支撑

**目标**：用真实评价增强观点的可信度

**操作**：

**AI 自动执行**：
1. 搜索包含"Claude Code"和"方便"的笔记
2. 提取真实评价："比ChatGPT方便，因为它能看到我的项目文件"


**目标**：用真实项目案例增强说服力

**操作**：

## 🎯 素材使用的 4 个原则

### 原则 1：真实第一

✅ **正确做法**：
- 使用真实的经历和评价
- 不夸大、不编造
- 保持客观和诚实

❌ **错误做法**：
- 编造不存在的经历
- 夸大效果和数据
- 伪装成他人经历

### 原则 2：自然融入

✅ **正确做法**：
- 改写成自然的长文逻辑
- 与上下文衔接流畅
- 符合文章整体风格

❌ **错误做法**：
- 直接复制粘贴原文
- 生硬插入不相关内容
- 与文章风格不一致

### 原则 3：适度使用

✅ **正确做法**：
- 每篇文章使用 2-3 个真实案例
- 选择最相关、最有说服力的素材
- 分散在文章不同位置

❌ **错误做法**：
- 堆砌大量素材
- 使用不相关的素材
- 集中在某一位置

### 原则 4：标注来源

✅ **正确做法**：
- 内在化表达："之前我记录过..."、"上周我测试了..."
- 隐性引用：自然融入，不刻意强调来源

❌ **错误做法**：
- 过度解释来源
- 显式标注："根据我的笔记..."

### 技巧 1：组合多个素材

**场景**：单个素材信息不够丰富

**方法**：

## ❓ 常见问题

### Q1: 素材不够怎么办？

**A**:
1. 从现在开始积累
2. 补记最近的重要经历
3. 可以使用品牌档案和历史内容作为补充

### Q2: 如何快速积累素材？

**A**:
1. 设置每天提醒（手机闹钟）
2. 养成习惯：做完一件事马上记
3. 周末回顾补记本周重要事项
4. 不强求每件事都记，记重要的

### Q3: 素材和文章主题不匹配怎么办？

**A**:
1. 搜索更宽泛的标签
2. 使用素材的思路而非完整内容
3. 如果实在没有，可以不用，不要强行使用

### Q4: 如何确保素材的隐私安全？

**A**:
1. 素材存储在本地，不上云
2. AI 读取素材时会标注来源
3. 你可以选择哪些素材可以使用
