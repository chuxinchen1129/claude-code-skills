# 微信文章创作工作流

## Overview

这是一个完整的内容创作系统，支持两大类型创作：

**自媒体写作**：
- 商业分析文章
- 科普文章

**品牌推广**：
- 公关软文（第三方媒体发表）
- 官方媒体发布
- 合作方自媒体发布
- 小红书品牌文案

**核心特点：**
- **分类专属工作流**：每种内容类型都有专门的流程
- **品牌档案管理**：结构化品牌信息库，确保信息准确一致
- **本地搜索能力**：智能检索本地品牌资料、历史内容、范文库 ✨
- **个人素材库集成** ✨ NEW：
  - Get 笔记手机记录，自动同步到本地
  - 按日期和主题双重索引，快速检索
  - 真实经历、观点、案例，大幅降低 AI 味
  - 自然融入文章，提升可信度和代入感
- **合作方风格学习**：实时分析并适配合作方表达风格
- **创作模板库**：6个专属模板，快速启动创作
- **半自动化流程**：关键节点等待你的确认

## 🎯 核心原则（不可妥协）⭐⭐⭐

### 1. Think Aloud 透明化思考 ⭐

**什么是 Think Aloud**：
- 每次重要决策都说明思考过程
- 让你理解 AI 为什么这样做
- 避免黑盒操作，提升可控性

**何时必须 Think Aloud**：
- ✅ **选题决策**：为什么提供这 3-4 个选题？
- ✅ **结构调整**：为什么选择这个文章结构？
- ✅ **内容删减**：为什么删除这部分内容？
- ✅ **风格调整**：为什么这样改写？
- ✅ **调研判断**：为什么认为这个信息源可靠？

**Think Aloud 格式**：
```
【思考过程】
决策：[做了什么决定]
理由：[为什么这样决定]
替代方案：[还有哪些可能的选择]
你的选择：[等待你的确认或调整]
```

**示例**：
```
【思考过程】
决策：我提供了 3 个选题方向（产品评测、使用心得、竞品对比）
理由：
- 用户需求是"写一篇关于 Claude Code 的文章"，但没有明确角度
- 产品评测：适合读者不了解产品的情况
- 使用心得：更真实、降低 AI 味、推荐
- 竞品对比：深度分析，但工作量较大

替代方案：
- 还可以提供"快速入门教程"、"高级技巧分享"等角度

你的选择：请告诉我你更倾向于哪个选题？或者你有其他想法？
```

**禁止项**：
- ❌ 不要说"我觉得"（要说"基于XX考虑"）
- ❌ 不要省略思考过程直接执行
- ❌ 不要用专业术语堆砌（要清晰易懂）

---

### 2. 选题讨论必做 ⭐

- **绝对禁止**：直接开始写文章！
- **必须执行**：提供 3-4 个选题，等待用户选择
- **参考**：`references/topic_frameworks.md`

### 3. 调研先行 ⭐

- **何时必须调研**：新技术、2024-2025 年新工具、不确定的细节
- **信息源优先级**：权威媒体 > 社区论坛 > 官方文档
- **禁止过时信息**：2025 年之前的信息禁止使用
- **参考**：`references/research_method.md`

### 4. 三遍审校机制 ⭐

- **第 1 遍**：内容审校（逻辑、事实、结构）
- **第 2 遍**：风格审校（AI 味降重）
- **第 3 遍**：细节打磨（标点、排版、节奏）
- **目标**：AI 检测率降至 30% 以下
- **参考**：`references/three_pass_review.md`

## 工作流一：自媒体写作（商业分析/科普）

**适用**：商业分析文章、科普文章
**参考资料**：sample_01-05, sample_07
**输出**：HTML格式长文

### 8阶段流程

**阶段1：信息搜集** ✨
- **本地优先搜索**（⚠️ 必须使用绝对路径）：
  - **技能资源目录**（绝对路径）：
    - 品牌档案：`~/.config/claude-code/skills/weChat-article-creator/assets/brand_profiles/`
    - 专业范文：`~/.config/claude-code/skills/weChat-article-creator/assets/samples/`
    - 创作模板：`~/.config/claude-code/skills/weChat-article-creator/assets/templates/`
    - 风格指南：`~/.config/claude-code/skills/weChat-article-creator/assets/user_style_analysis.md`
    - 历史输出：`~/.config/claude-code/skills/weChat-article-creator/assets/output/`
  - **个人素材库搜索** ✨ NEW：
    - 在 `~/Desktop/个人素材库/主题索引/` 中搜索相关标签
    - 在 `~/Desktop/个人素材库/按日期记录/` 中查找近期笔记
    - 提取真实经历、观点、案例用于降AI味
  - **用户已发布文章库**（Obsidian）：
    - 位置：`~/Library/Mobile Documents/com~apple~CloudDocs/ObsidianVault-2026/01内容创作/已发布/`
    - 包含：宠商圈（25+篇）、悟昕（13+篇）等真实发布文章
- **网络补充搜索**（如需要）：
  - 搜索最新数据和行业动态
  - 补充真实案例和专家观点
  - 聚焦具体场景和可复用案例细节
- **信息整合**：
  - 合并本地和网络搜索结果
  - 标注信息来源和可信度
  - 优先使用个人素材库中的真实内容替代编造案例

**阶段2：讨论整合**
- 整合所有信息源
- 与你讨论想法和切入点

**阶段3：选题规划**
- 根据样本风格选择策略（商业分析/科普清单/科学拆解）
- 输出备选标题

**阶段4：文章撰写**
- 参考user_style_analysis.md中的对应风格
- 按照结构模板撰写

**阶段5：风格润色** ⭐⭐⭐ 优化版
- **步骤1**：深刻阅读范文（`references/writing_style_optimization.md`）
  - 逐句分析范文特征
  - 提取可复用模板
  - 对比AI生成与范文的差距
- **步骤2**：应用 8 大优化维度
  1. 少用引号：强调不用、概念不用、对话省略、观点转述
  2. 多用人话：用"我觉得"、用"你"、替换书面词、长句拆短
  3. 表达丰富：避免重复句式、减少"的"字、句式交替
  4. 故事性：SCQA框架、细节公式、让读者在场
  5. Facts & Figures：数据密度（每1-2段）、类型多样、案例具体
  6. 节奏感：长短句配比（短30%、中50%、长20%）、节奏变化
- **步骤3**：去 AI 味处理
  - 删除套话
  - 拆解 AI 句式
  - 加入真实细节
  - 加入个人态度

**阶段6：标题优化**
- 优化大小标题
- 输出引言

**阶段7：配图**
- Unsplash搜索下载
- 按顺序命名

**阶段8：HTML排版** ⭐⭐⭐ 更新版
- **步骤1**：判断文章类型
  - 宠商圈周报：标题包含"周报"+"宠商圈"，或用户明确说明
  - 普通文章：所有其他文章（商业分析、科普、品牌故事等）
- **步骤2**：应用对应排版规范（`references/html_formatting_guide.md`）

  **宠商圈周报排版**：
  - 使用 `.weekly-report` 容器
  - 01/02/03 标题使用渐变背景
  - 副标题有左边框
  - 内容卡片有边框和背景
  - 数据卡片有独立样式

  **普通文章排版**：
  - 使用 `.article-content` 容器
  - 01/02/03 标题使用渐变背景（仅这些）
  - 其他标题只用橙色加粗，无边框、无背景
  - 正文无边框
  - 数据展示简洁样式

- **步骤3**：生成最终 HTML

## 工作流二：公关软文（第三方发表）

**适用**：第三方媒体发布的品牌软文
**参考资料**：sample_08
**输出**：Word/HTML格式
**约束**：严格遵循品牌信息，不自由发挥

### 8阶段流程

**阶段1：获取品牌档案**
- 读取brand_profiles/[品牌名].md
- 确认核心卖点（不可更改）
- 确认品牌调性要求

**阶段2：确定新闻切入点**
- 你选择：行业热点 / 产品上市 / 品牌活动
- 提供新闻背景信息

**阶段3：信息整合**
- 新闻背景 + 品牌信息
- 严格遵循你提供的关键信息
- 不添加未经确认的内容

**阶段4：文章结构设计**
- 新闻性开头（300字内）
- 问题分析/趋势解读
- 自然过渡到品牌
- 品牌解决方案介绍
- 行业价值/用户价值

**阶段5：内容撰写**
- 参考sample_08科学拆解型结构
- 严格遵循品牌档案核心卖点
- 每个品牌点需有数据或案例支撑
- 保持客观第三方口吻

**阶段6：品牌信息核对**
- 与品牌档案逐项核对
- 确认无违规内容
- 确认核心卖点完整呈现

**阶段7：标题优化**
- 新闻式标题（避免过于营销化）
- 主标题 + 副标题

**阶段8：格式输出**
- Word格式（便于投稿）
- 或HTML格式

**关键检查点**：
- [ ] 所有品牌信息有来源依据
- [ ] 无夸大或未经证实的内容
- [ ] 新闻性与品牌推广平衡

## 工作流四：合作方自媒体发布

**适用**：在合作方自媒体平台发布
**参考资料**：需要实时学习合作方风格
**输出**：符合合作方格式要求
**约束**：品牌信息 + 合作方风格融合

### 9阶段流程（增加风格学习）

**阶段1：获取品牌档案**
- 读取brand_profiles/[品牌名].md
- 确认本次推广核心信息

**阶段2：合作方信息提供**
- 你提供：合作方自媒体账号/链接/名称
- 或提供3-5篇该自媒体过往文章

**阶段3：合作方风格实时分析** ✨
- 分析该自媒体的：
  * 开头方式（对话/引用/故事）
  * 语言风格（严肃/活泼/专业）
  * 段落结构（长短/分段）
  * 标题习惯（长度、用词）
  * emoji使用（频率、类型）
  * 特殊标签或格式
- 生成风格分析报告
- 可选择保存到partner_analysis/[合作方名]/

**阶段4：确定切入点**
- 结合合作方风格特点
- 选择合适的内容角度
- 确认品牌信息嵌入点

**阶段5：内容撰写**
- 应用合作方风格
- 融入品牌信息
- 平衡双方诉求

**阶段6：风格适配检查**
- 对照合作方风格分析
- 确认品牌信息完整
- 调整不匹配部分

**阶段7：品牌信息核对**
- 与品牌档案核对
- 确认核心卖点

**阶段8：格式适配**
- 符合合作方平台要求
- 长度、格式、配图

**阶段9：输出交付**
- 按合作方要求格式
- 备注品牌信息位置

**关键检查点**：
- [ ] 合作方风格匹配度
- [ ] 品牌信息完整性
- [ ] 两者平衡度
