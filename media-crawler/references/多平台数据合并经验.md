# 多平台数据合并经验复盘 (2026-02-03)

> **问题**: 不同社交媒体平台的数据结构不同，直接合并导致数据丢失
> **案例**: 李宁奥运项目数据合并

---

## ⚠️ 关键原则

> **合并多平台数据前，必须先检查每个平台的JSON结构，确认字段名**

---

## ❌ 典型错误案例

### 错误代码

```python
# 统一使用 video_url
video_url = item.get('video_url', '')
```

### 结果

| 平台 | 结果 | 原因 |
|------|------|------|
| 小红书 | ✅ 正常（11个视频） | 使用 `video_url` |
| 抖音 | ❌ 0个视频（27个全部丢失） | 使用 `video_download_url` |

---

## ✅ 正确做法

### 步骤1：检查字段结构

```python
import json

# 读取抖音数据
with open('data/douyin/json/search_contents_2026-02-03.json', 'r') as f:
    douyin_data = json.load(f)

# 打印所有字段
print(list(douyin_data[0].keys()))
# 输出: ['aweme_id', 'aweme_type', ..., 'video_download_url', ...]
```

### 步骤2：针对不同平台使用不同字段

```python
def process_xhs_data(item):
    return {
        '视频URL': item.get('video_url', ''),  # 小红书用 video_url
        ...
    }

def process_douyin_data(item):
    return {
        '视频URL': item.get('video_download_url', ''),  # 抖音用 video_download_url
        ...
    }
```

---

## 📊 常见字段名差异

| 字段用途 | 小红书 | 抖音 |
|---------|--------|------|
| 内容ID | `note_id` | `aweme_id` |
| 视频URL | `video_url` | `video_download_url` |
| 作者ID | `user_id` | `sec_uid` |
| 内容URL | `note_url` | `aweme_url` |
