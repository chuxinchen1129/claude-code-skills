## 飞书配置

### 方式一：使用 Webhook（推荐）

1. **创建飞书机器人**
   - 在飞书群聊中添加自定义机器人
   - 选择"自定义机器人"
   - 复制 Webhook URL

2. **配置 Webhook**

创建 `config/feishu.json`:
```json
{
  "webhook_url": "https://open.feishu.cn/open-apis/bot/v2/hook/xxxxxxxx"
}
```

或设置环境变量:
```bash
export FEISHU_BOT_WEBHOOK="https://open.feishu.cn/open-apis/bot/v2/hook/xxxxxxxx"
```

### 方式二：使用应用凭据

1. **创建飞书应用**
   - 访问 [飞书开放平台](https://open.feishu.cn/)
   - 创建企业自建应用
   - 获取 App ID 和 App Secret

2. **配置权限**
   - 在应用权限中添加：
     - `im:message` - 发送消息权限
     - `im:message:send_as_bot` - 以机器人身份发送

3. **配置凭据**

创建 `scripts/config.py`:
```python
FEISHU_APP_ID = "cli_xxxxxxxxxxxxx"
FEISHU_APP_SECRET = "xxxxxxxxxxxxxxxxxxxx"
```

或设置环境变量:
```bash
export FEISHU_APP_ID="cli_xxxxxxxxxxxxx"
export FEISHU_APP_SECRET="xxxxxxxxxxxxxxxxxxxx"
```

4. **获取接收者信息**

需要发送消息给用户或群聊，需要获取对应的 `open_id` 或 `chat_id`:
```python
# 可以通过飞书 API 查询
# 或在飞书中查看群聊信息
CHAT_ID = "oc_xxxxxxxxxxxxxxxx"
```

## 监控配置

### brands.json 配置

位置: `config/brands.json`

```json
{
  "keywords": ["悟昕睡眠仪", "悟昕 睡眠仪", "悟昕助眠"],
  "competitors": ["强脑", "左点"],
  "interval_hours": 2,
  "max_notes": 10,
  "screenshot_enabled": true,
  "comment_min_length": 10,
  "alert_on_brand_mention": true
}
```

### 配置项说明

| 配置项 | 类型 | 说明 | 默认值 |
|--------|------|------|--------|
| `keywords` | array | 搜索关键词列表 | - |
| `competitors` | array | 竞品品牌列表 | - |
| `interval_hours` | number | 扫描间隔（小时） | 2 |
| `max_notes` | number | 每次扫描最大笔记数 | 10 |
| `screenshot_enabled` | boolean | 是否启用截图 | true |
| `comment_min_length` | number | 评论最小长度（字） | 10 |
| `alert_on_brand_mention` | boolean | 发现品牌立即预警 | true |

## 运行监控

### 单次扫描

```bash
cd ~/.claude/skills/competitor-alert/scripts
python monitor.py --once
```

### 启动守护进程

```bash
# 前台运行（测试用）
python monitor.py --daemon

# 后台运行（生产用）
python monitor.py --start
```

### 进程管理

```bash
# 查看状态
python monitor.py --status

# 停止监控
python monitor.py --stop

# 重启监控
python monitor.py --restart
```

### 设置开机自启

#### macOS (使用 launchd)

创建 `~/Library/LaunchAgents/com.competitor-alert.plist`:

```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
    <key>Label</key>
    <string>com.competitor-alert</string>
    <key>ProgramArguments</key>
    <array>
        <string>/usr/bin/python3</string>
        <string>/Users/echochen/.claude/skills/competitor-alert/scripts/monitor.py</string>
        <string>--daemon</string>
    </array>
    <key>RunAtLoad</key>
    <true/>
    <key>KeepAlive</key>
    <true/>
    <key>StandardOutPath</key>
    <string>/Users/echochen/.claude/skills/competitor-alert/scripts/monitor.log</string>
    <key>StandardErrorPath</key>
    <string>/Users/echochen/.claude/skills/competitor-alert/scripts/monitor_error.log</string>
</dict>
</plist>
```

加载服务:
```bash
launchctl load ~/Library/LaunchAgents/com.competitor-alert.plist
```

#### Linux (使用 systemd)

创建 `/etc/systemd/system/competitor-alert.service`:

```ini
[Unit]
Description=Competitor Alert Monitor
After=network.target

[Service]
Type=simple
User=echochen
WorkingDirectory=/home/echochen/.claude/skills/competitor-alert/scripts
ExecStart=/usr/bin/python3 /home/echochen/.claude/skills/competitor-alert/scripts/monitor.py --daemon
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
```

启动服务:
```bash
sudo systemctl daemon-reload
sudo systemctl enable competitor-alert
sudo systemctl start competitor-alert
```

## 故障排查

### 问题：没有收到飞书通知

**检查清单:**
1. ✅ 确认飞书 Webhook URL 正确
2. ✅ 确认网络连接正常
3. ✅ 查看日志文件: `logs/monitor_*.log`
4. ✅ 测试飞书通知:
   ```bash
   python send_feishu.py --test-alert
   ```

### 问题：采集数据失败

**可能原因:**
- MediaCrawler 未正确配置
- 小红书账号未登录
- 网络连接问题

**解决方法:**
1. 检查 MediaCrawler 配置
2. 手动登录小红书账号
3. 查看 MediaCrawler 日志

### 问题：监控进程异常退出

**检查方法:**
```bash
# 查看错误日志
tail -50 logs/monitor_error.log

# 检查进程状态
python monitor.py --status
```

**常见原因:**
- 配置文件格式错误
- 依赖包未安装
- 权限问题
